# ハイライト矩形のセル整列問題

検索ハイライトの矩形は Kitty Graphics Protocol (KGP) のオーバーレイ配置で描画される。
本ドキュメントでは KGP のサブセルオフセットの実測挙動、セル丸めの影響、
および現状の実装と改善方針を記録する。

## KGP 配置パラメータの実測挙動

### 実験環境

- ターミナル: Ghostty
- セルサイズ: 13×28 px (cw=13, ch=28)
- テスト画像: 100×24 (赤), 100×28 (青, =ch), 100×1 (緑), 200×24 (橙)
- 実験スクリプト: [`tools/kgp_offset_experiment.py`](../tools/kgp_offset_experiment.py)

### 公式仕様の記述

Kitty Graphics Protocol 仕様より:

> "You can also specify extra `X=3` and `Y=4` pixel offsets to display from
> a different origin within the cell. Note that the offsets must be smaller
> than the size of the cell."

> "Note that if you specify a start cell offset via the `X,Y` keys,
> it is not added to the number of rows/columns."

### 実測で判明したモデル

```
scaling_target = c * cw × r * ch          ← X, Y に無関係
render:  source(w, h) → scale → scaling_target
display: scaling_target を (X, Y) だけシフトし、セル境界でクリップ
visible: (c*cw - X) × (r*ch - Y)          ← クリップ後の可視領域
```

**旧ドキュメントの誤り**: `visible = r*ch - Y` を「スケーリングターゲットが縮小する」
と解釈していたが、実際には**スケーリングターゲットは `r*ch` で固定**。
Y はレンダリング後の純粋な位置シフトであり、スケーリングには影響しない。

### 実験結果の詳細

#### Exp 1: 100×24 red PNG, r=1, varying Y (ch=28)

24px の PNG が r=1 で 28px にスケールアップされ、Y オフセットで下方向にシフト。

```
Y= 0: |████████████████████████████| 28px 表示 (24→28 にスケール)
Y= 2: __|██████████████████████████| 26px 可視 (2px シフト, 下端 2px クリップ)
Y= 4: ____|████████████████████████| 24px 可視
Y= 8: ________|████████████████████| 20px 可視
Y=14: ______________|██████████████| 14px 可視
```

観察:
- Y=0 では 24px→28px のスケーリングが発生（セル全体を埋める）
- Y を増やすとブロックが下にシフトし、**下端がセル境界でクリップ**される
- スケーリング後の画像自体の見た目は Y に関係なく同一

#### Exp 2: 100×24 red PNG, r=2, varying Y (ch=28)

r=2 で 24px→56px にスケール。Y でシフト+クリップ。

```
Y= 0: |████████████████████████████████████████████████████████| 56px
Y= 4: ____|████████████████████████████████████████████████████| 52px
Y=14: ______________|██████████████████████████████████████████| 42px
Y=27: ___________________________|█████████████████████████████| 29px
```

観察:
- r=2 では Y=27 (ch-1) でも 29px 可視で、r=1 より余裕がある
- スケーリングターゲット (56px) は Y によらず一定

#### Exp 3: 100×28 blue PNG (=ch), r=1, varying Y

ソースがセル高さに一致するため、スケーリングなし (1:1) での純粋なオフセット挙動。

```
Y=0: フルセル高さ表示
Y=2: 2px 下にシフト、下端 2px クリップ
Y=4: 4px 下にシフト、下端 4px クリップ
Y=8: 8px 下にシフト、下端 8px クリップ
```

観察: スケーリングが介在しない環境でも同じシフト+クリップ挙動を確認。

#### Exp 4: 100×1 green PNG — 極端なスケーリング

```
r=1 Y=0: 1px → 28px にスケール (セル全体を緑で埋める)
r=1 Y=4: 同じスケール、4px 下にシフト
r=2 Y=0: 1px → 56px にスケール (2 セル分)
```

観察: ソースサイズに関係なく、スケーリングターゲットは常に `r*ch`。

#### Exp 5: 100×24 red PNG, r=1, varying source h

```
h=24: ソース全体 (24px) → 28px にスケール
h=12: ソース上半分 (12px) → 28px にスケール ← 同じ表示高さ!
h= 6: ソース上部 (6px)  → 28px にスケール ← 同じ表示高さ!
h= 1: ソース 1px        → 28px にスケール ← 同じ表示高さ!
```

観察: ソース矩形 `h` はクロップのみ。表示サイズは `r*ch` で固定。
`h` でスケーリングを制御する方法は存在しない。

#### Exp 7: X オフセット (200×24 orange PNG, cw=13)

```
X= 0: 左端から表示
X= 4: 4px 右にシフト、右端クリップ
X= 6: 6px 右にシフト
X=12: 12px 右にシフト (ほぼ 1 セル分)
```

観察: Y と同じモデル — 水平方向のシフト + 右端クリップ。

### モデルの要約

| パラメータ | 役割                               | スケーリングへの影響 |
|-----------|------------------------------------|--------------------|
| `w`, `h`  | ソース矩形のクロップ                 | なし (クロップ後スケール) |
| `c`, `r`  | 表示セル数 → スケーリングターゲット    | `c*cw × r*ch`     |
| `X`, `Y`  | セル内ピクセルオフセット (シフト+クリップ) | **なし**          |

**重要**: `X`/`Y` はスケーリングに影響しない。位置精度とサイズ精度のトレードオフは
存在しない。Y オフセットは安全にピクセル精度の位置合わせに使える。

## 現在の実装: Y サブセルオフセット + 部分透過パターン

実験で Y オフセットがスケーリングに影響しないことが判明したため、
Y サブセルオフセットによるピクセル精度の垂直位置合わせを採用した。
セル境界をまたぐオーバーフロー領域は部分透過パターンで 2 回配置する。

### アーキテクチャ

```
highlight.rs::find_highlight_rects()
  → Frame ツリーを走査、正規表現マッチ → Vec<HighlightRect> {x_px, y_px, w_px, h_px}

terminal.rs::place_rects_in_region()
  → HighlightRect → KGP 配置コマンド (最大 2 回/rect)
  → 行決定: 上端方式 screen_y_px / ch (旧: 中点方式)
  → X: サブセルオフセット使用 (ピクセル精度, 変更なし)
  → Y: サブセルオフセット使用 (ピクセル精度, ★新規)
  → オーバーフロー: 部分透過 1×24 RGBA パターンで次行をカバー
```

### 使用画像

| 用途 | ID フィールド | フォーマット | サイズ | 説明 |
|------|------------|-----------|-------|------|
| 主配置 | `full_id` | PNG (f=100) | 2048×24 | 幅クロップ用。既存 `assets/highlight.png` |
| 75% カバー | `p75_id` | raw RGBA (f=32) | 1×24 | 上 18 行が半透明黄、下 6 行が透明 |
| 50% カバー | `p50_id` | raw RGBA (f=32) | 1×24 | 上 12 行が半透明黄、下 12 行が透明 |
| 25% カバー | `p25_id` | raw RGBA (f=32) | 1×24 | 上 6 行が半透明黄、下 18 行が透明 |

4 画像すべて `HighlightImages` 構造体で管理し、初回使用時に一括アップロード。
raw RGBA は 96 bytes → base64 128 bytes で 1 チャンクに収まる。

### Y 配置ロジック

```rust
// 上端方式: top px からセル行とサブセルオフセットを算出
let screen_y_px = r.y_px.saturating_sub(rgn.src_y);
let row = rgn.screen_row + (screen_y_px / rgn.ch) as u16;
let y_off = screen_y_px % rgn.ch;

// 1st placement: FULL 画像 + Y サブセルオフセット
// KGP: Y=y_off でセル内の正確な位置にシフト、下端はセル境界でクリップ
write!("...i={full_id},w={src_w},h=24,X={x_off},Y={y_off},c={cols},r=1,...")?;

// 2nd placement: オーバーフローがあれば次行に部分パターン
let first_coverage = ch - y_off;  // 1st セルでカバーされるピクセル数
if h_px > first_coverage {
    let overflow = (h_px - first_coverage).min(ch);
    let pattern = select_overflow_pattern(overflow, ch);
    // pattern に応じて P25/P50/P75/FULL を次行に配置
}
```

### パターン選択ロジック

`select_overflow_pattern(overflow_px, ch)` でオーバーフロー量に応じた
パターンを選択:

| overflow / ch | パターン | 充填行 (of 24) | 備考 |
|--------------|---------|---------------|------|
| ≤ 25% | P25 | 6 | 軽いカバレッジ |
| ≤ 50% | P50 | 12 | |
| ≤ 75% | P75 | 18 | |
| > 75% | Full | 24 (2048×24 PNG) | セル全体をカバー |

1×24 のパターンは `c={cols}` で水平方向にスケールされるが、
ソリッドカラーなので 1px→任意幅のスケールは視覚的に同一。

### カバレッジ精度 (ch=28, h_px=24 の例)

| y_off | overflow | pattern | 実表示 | 最大超過 |
|-------|----------|---------|-------|---------|
| 0–4   | なし     | —       | 1 セル内で完結 | — |
| 5–11  | 1–7      | P25     | ~7px  | ~6px |
| 12–18 | 8–14     | P50     | ~14px | ~6px |
| 19–25 | 15–21    | P75     | ~21px | ~6px |
| 26–27 | 22–23    | FULL    | 28px  | ~6px |

最大超過は約 6px の半透明黄色で、視覚的に許容範囲。

### 旧方式との比較

| 項目 | 旧 (中点方式) | 新 (Y オフセット + 部分透過) |
|------|-------------|--------------------------|
| 行決定 | `(screen_y_px + h_px/2) / ch` | `screen_y_px / ch` |
| Y サブセルオフセット | 未使用 | `screen_y_px % ch` |
| 垂直精度 | 最大 ch-1 px のズレ | ピクセル精度 |
| KGP 配置数/rect | 1 | 最大 2 |
| 画像数 | 1 (2048×24 PNG) | 4 (PNG + 3 RGBA パターン) |
| r=2 フォールバック | 不要 (r=1 固定) | 不要 (r=1 固定 + 2nd placement) |

r=1→2 の不連続ジャンプ問題を回避しつつ、ピクセル精度の位置合わせを実現。

### 設計判断

1. **r=2 ではなく 2 回配置**: r=2 にするとスケーリングターゲットが 2*ch に跳び、
   画像が倍の高さに拡大される不連続性がある。2 回の r=1 配置で回避。

2. **部分透過パターン**: オーバーフロー領域は常にセル全体をカバーすると
   過剰な黄色が出る。4 段階の部分透過で超過を最大 ~6px に抑制。

3. **`const fn` でコンパイル時生成**: パターンは `make_pattern()` で
   コンパイル時に生成。ランタイムコストゼロ。

4. **raw RGBA (f=32)**: 1×24 の小さなパターンは PNG エンコードするより
   raw RGBA の方がシンプル。96 bytes で 1 チャンク送信。

## 実験スクリプト

全実験コードは [`tools/kgp_offset_experiment.py`](../tools/kgp_offset_experiment.py) にある。
KGP 対応ターミナル (Kitty / Ghostty / WezTerm) で実行可能。

```bash
python3 tools/kgp_offset_experiment.py
```

8 つの実験で X/Y オフセット、r パラメータ、ソース矩形 h の挙動を
テキスト参照付きで視覚的に検証する。
