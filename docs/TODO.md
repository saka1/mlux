## ターミナル E2E テスト（KGP シミュレータ）

- 背景: 検索モード→Normal復帰時に画面が戻らないバグがあった。Effect層のロジックは正しいが、
  ターミナル I/O（画像の削除・再配置・テキストクリア）の組み合わせで発生。ユニットテストでは捕まらない。
- 方針: KGP のサブセット（5コマンド: `a=t`, `a=p`, `a=d,d=i/I/A`）の状態トラッカーを作り、
  「Confirm 後に placements が非空か」等をアサートする E2E テスト。
- ステップ:
  1. `terminal.rs` の全関数を `&mut impl Write` 受けに抽象化（最重量だがログ/リプレイにも使える）
  2. 出力バイト列から APC (`\x1b_G...\x1b\\`) をパースする KGP ステートトラッカー（100-200行）
  3. 入力側は `Vec<Event>` を食わせるアダプタ
- KGP パーサ自体は小さい。本当に重いのは stdout/stdin の分離リファクタ。

## 画像サイズ制御
現状 `width: 100%` で常に全幅表示。小さい画像（例: 1x1px PNG）も引き伸ばされる。
理想は CSS `max-width: 100%` 相当（元サイズ以上に拡大しない）だが、Typst 0.14 では直接表現不可。
案: PNG/JPEG ヘッダから実ピクセル幅を読み取り、PPI換算でコンテンツ幅より小さければ `width` 指定なし、大きければ `width: 100%` にする。使い勝手を見つつ判断。

## リモート画像: スキーム検証の大文字小文字対応

`image.rs` の URL 判定は `starts_with("http://")` で case-sensitive。
`Http://` や `HTTPS://` は URL として認識されずローカルパス扱いになる。
pulldown-cmark が大文字スキームのリンクをパースしない可能性が高く実害は低いが、
防御的に `to_ascii_lowercase()` で正規化すべき。
また `://` を含む未知スキーム（`ftp://`, `file://` 等）も明示的に拒否する。

## Sandbox 失敗時のユーザー通知

`fork_render/mod.rs` で `enforce_read_only_sandbox()` が失敗した場合、
`log::warn!` のみで処理を続行する。`--log` 未指定時はユーザーに一切通知されない。
サンドボックスなしで動作していることをユーザーが認識できるよう、
少なくとも stderr に警告を出すか、`--no-sandbox` 未指定時は致命的エラーにすることを検討。
