# mlux 完成度評価 — less / glow / 他ツールとの詳細比較

2026-03-21 時点（v1.12.0）

## 根本的な設計思想の違い

まず認識すべきは、mlux が他のどのツールとも**根本的に異なるアプローチ**を取っていることである。

| ツール | レンダリング方式 | 出力 |
|--------|------------------|------|
| **less** | レンダリングなし（生テキスト） | 文字セル |
| **glow** | ANSI エスケープコード | 文字セル（装飾付き） |
| **bat** | シンタックスハイライト | 文字セル（装飾付き） |
| **mdcat** | ANSI + 画像プロトコル | 文字セル + インライン画像 |
| **mlux** | **Typst 組版エンジン → PNG** | **ピクセルレンダリング** |

glow や bat が「ターミナルの文字セルをどう装飾するか」という問題を解いているのに対し、mlux は「プロフェッショナルな組版結果をターミナルに表示する」という、一段上の問題を解いている。これは PDF ビューアに近い。

---

## 各ツールの概要

### less (Unix pager)

- レンダリングなし。Markdown をプレーンテキストとして表示
- 双方向スクロール、正規表現検索（`/`, `?`, `&`）、マーク、複数ファイル対応
- `F` キーで `tail -f` 相当のフォローモード
- パイプ入力 (`command | less`) に完全対応
- LESSOPEN/LESSCLOSE による入力プリプロセッサ拡張
- ほぼ全てのターミナルで動作。数十年の安定性

### glow (charmbracelet/glow)

- goldmark (CommonMark 準拠) でパース → Glamour で ANSI レンダリング
- **対応要素:** 見出し、段落、強調、取り消し線、インラインコード、フェンスコードブロック（chroma によるシンタックスハイライト）、引用、リスト（タスクリスト含む）、テーブル、リンク、水平線、定義リスト
- **非対応:** 画像（alt テキストのみ表示）、数式/LaTeX、Mermaid
- TUI モード（ファイルブラウザ）+ ページャモード
- `/` で前方検索、`n`/`N` で結果巡回
- テーマ: `ascii`, `auto`, `dark`, `dracula`, `tokyo-night`, `light`, `notty`, `pink` + カスタム JSON スタイルシート
- ANSI のみ（Kitty/Sixel 不要）、ほぼ全ターミナルで動作

### bat (sharkdp/bat)

- Markdown をレンダリングしない。ソースコードとしてシンタックスハイライト表示
- syntect (Sublime Text 定義) による 166+ 言語対応
- 行番号、Git diff インジケータ、自動ページング
- less をバックエンドページャとして使用可能

### mdcat (swsnr/mdcat) — 2025年1月アーカイブ済み

- ANSI レンダリング + iTerm2/Kitty/Sixel による画像インライン表示
- OSC 8 ハイパーリンク、シンタックスハイライト
- ページャ機能なし（`cat` 相当、パイプで less に渡す）
- **メンテナンス終了**

### mcat (Skardyy/mcat)

- 画像・動画・PDF・DOCX・Markdown のマルチフォーマットビューア
- Sixel / Kitty Graphics Protocol による画像表示
- 比較的新しいプロジェクト

## mlux の現在の機能セット（2026-03-21 時点）

### Markdown 変換 (pipeline/convert.rs)

**対応ブロック要素:**
- 見出し (h1-h6、全レベル styled)
- 段落（ソフト改行・ハード改行）
- フェンスコードブロック（言語指定付き、シンタックスハイライト対応）
- 順序なしリスト / 順序付きリスト（ネスト対応）
- 引用（ネスト対応、深さ上限 10）
- テーブル（列数・セル内容・アライメント）
- 水平線
- **画像** (`![alt](path)`) — ローカル画像をインライン表示
- **数式** — インライン (`$...$`) / ディスプレイ (`$$...$$`)、mitex 経由で Typst 数式に変換
- **Mermaid ダイアグラム** — ` ```mermaid ` ブロックを SVG にレンダリングしてインライン埋め込み

**対応インライン要素:**
- 太字、イタリック、取り消し線
- インラインコード（バッククォート含むケースは `#raw()` 関数使用）
- リンク（空 URL は平文に降格）
- 画像（`![alt](path)`）
- インライン数式 (`$...$`)
- Typst 特殊文字エスケープ (`#*_<>@$\\/~()[]`)

**非対応:**
- タスクリスト / チェックボックス
- 脚注
- 定義リスト
- 上付き / 下付き文字
- インライン HTML（保存はされるが表示に反映されない）
- YAML フロントマター

### レンダリング (pipeline/)

- Typst 0.14 組版エンジンによる PDF 品質のレンダリング
- 設定可能パラメータ: width (660pt), PPI (144), tile-height (500pt), theme
- フォント: IPAGothic → Noto Sans CJK JP → Noto Sans
- テーマ: Typst ソースファイル（Catppuccin Mocha / Catppuccin Latte 同梱）
- **テーマ自動検出** (`--theme auto`): ターミナルの背景色をクエリし、dark/light を自動選択

### 画像サポート

- **ローカル画像**: 入力ファイルからの相対パスで読み込み（PNG, JPEG, SVG 等）
- **リモート画像**: `--allow-remote-images` フラグで HTTP/HTTPS 画像のフェッチを許可（デフォルト無効）
- **セキュリティ**: 絶対パス拒否、ベースディレクトリ外拒否、symlink 検証、50MB サイズ制限、data URL 拒否
- Typst の `#image()` 関数でネイティブ表示 — ピクセル精度のレイアウト

### 数式レンダリング

- **インライン数式**: `$...$` → mitex で LaTeX → Typst 数式に変換
- **ディスプレイ数式**: `$$...$$` → 中央寄せブロック数式
- mitex 変換失敗時は元の LaTeX テキストにフォールバック
- **他のどのターミナル Markdown ビューアも実現していない機能** — glow, bat, mdcat いずれも数式非対応

### Mermaid ダイアグラム

- ` ```mermaid ` フェンスコードブロックを自動検出
- mermaid-rs-renderer で SVG にレンダリング
- SVG を Typst の `#image()` でインライン表示
- mermaid-rs v0.2.0 の SVG バグ（font-family 属性のエスケープ漏れ）に対するワークアラウンド内蔵

### シンタックスハイライト

- **tmTheme ベースのシンタックスハイライト**
- Catppuccin Mocha / Latte 用 tmTheme ファイルをバーチャルファイルとして埋め込み
- Typst の `#set raw(theme: ...)` 機能を活用
- 言語指定付きコードブロック（` ```rust ` 等）で意味ベースのカラーリングが実現
- glow（chroma）・bat（syntect）と並ぶ品質。さらにピクセル精度の色表現が可能

### ターミナルビューア (viewer/)

**表示:**
- Kitty Graphics Protocol によるピクセルレンダリング
- 6 カラムサイドバーに行番号表示
- ステータスバー（ファイル名、スクロール位置、パーセンテージ、フラッシュメッセージ）
- 非同期ビルド + ローディング画面（100ms 以上かかる場合）、ビルド中断対応（`q`/`Esc`/`Ctrl-C`）

**モード構成（6モード）:**

```
Normal ─(/)→ Search ─(Enter)→ Normal
       ─(:)→ Command ─(Enter/Esc)→ Normal
       ─(O)→ URL Picker ─(Enter/Esc)→ Normal
       ─(t)→ TOC ─(Enter/Esc)→ Normal
       ─(:log)→ Log Viewer ─(q/Esc)→ Normal
```

**ナビゲーション (vim/less ライク — Normal モード):**

| キー | 動作 |
|------|------|
| `j` / `↓` | 下スクロール（`Nj` で N ステップ） |
| `k` / `↑` | 上スクロール（`Nk` で N ステップ） |
| `d` | 半ページ下（`Nd` 対応） |
| `u` | 半ページ上（`Nu` 対応） |
| `g` | 先頭へ / `Ng` で N 行目へジャンプ |
| `G` | 末尾へ / `NG` で N 行目へジャンプ |
| `y` | 行ヤンク（`Ny` で N 行目のコードをコピー） |
| `Y` | ブロックヤンク（`NY` で N 番目のブロック全体をコピー） |
| `/` | 検索モードへ |
| `n` / `N` | 次 / 前の検索一致へジャンプ |
| `o` | 行の URL をブラウザで開く / ローカル .md はタグジャンプ（`No` で N 行目） |
| `O` | URL ピッカーモードへ（全 URL 一覧） |
| `Ctrl-O` | 戻る（タグジャンプスタック） |
| `t` | 目次（TOC）モードへ |
| `:` | コマンドモードへ |
| `Esc` | 数値入力キャンセル |
| `q` / `Ctrl+C` | 終了 |

**検索モード:**
- `/` で起動
- Markdown ソースに対する正規表現検索（スマートケース: 全小文字なら大文字小文字無視）
- `↓`/`↑` で結果巡回、`Alt+1-9` で N 番目の一致に直接ジャンプ
- `Enter` で該当行へジャンプし Normal モードに戻る
- **ワードレベルの検索ハイライト** — ContentIndex による双方向マッピング（Markdown バイト → フレーム座標）で、一致箇所を KGP オーバーレイ矩形として描画。アクティブな一致はオレンジ色で強調
- Normal モードから `n`/`N` で次/前の一致へジャンプ（ハイライト維持）

**目次（TOC）モード:**
- `t` で起動
- ATX 見出し（`#` 〜 `######`）を一覧表示（コードブロック内は除外）
- `j`/`k` で選択、`Enter` で該当見出しへジャンプ
- 見出しレベルのインデント表示 + Markdown 行番号

**URL ピッカーモード:**
- `O` または `:open` で起動
- 文書内の全 URL を一覧表示（Markdown リンクテキスト付き）
- 裸の `https://` URL も自動抽出
- `j`/`k` で選択、`Enter` でブラウザ起動

**リンクナビゲーション（タグジャンプ）:**
- `No` でローカル `.md` ファイルへのリンクを踏むと、そのファイルにジャンプ
- `Ctrl-O` で前のファイル + スクロール位置に戻る（LIFO ジャンプスタック）
- ジャンプスタックはリビルド・リサイズを跨いで保持

**コマンドモード:**
- `:reload` / `:rel` — 設定とドキュメントを再読み込み
- `:quit` / `:q` — 終了
- `:open` — URL ピッカーモードへ
- `:back` — 戻る（`Ctrl-O` と同等）
- `:log` — ログビューアモードへ

**ログビューアモード:**
- `:log` で起動（`--log` または `--debug` フラグ使用時に有効）
- リングバッファ（1024 エントリ、FIFO オーバーフロー）に蓄積されたログを表示
- ログレベル別カラー表示（ERROR=赤, WARN=黄, INFO=標準, DEBUG/TRACE=dim）
- `/` でログ内正規表現検索、`n`/`N` で結果巡回
- `y` で全エントリをクリップボードにコピー
- `j`/`k` スクロール、`g`/`G` 先頭/末尾

**アーキテクチャ:**
- タイルベース遅延レンダリング（ドキュメント全体を 1 回コンパイル → フレームツリーを垂直タイルに分割）
- 可視タイルのみ PNG 化、LRU キャッシュ（前後 ±4 タイル、コンテンツアドレスでリビルド跨ぎ）
- プリフェッチワーカー（前方 2 + 後方 1 タイル先行レンダリング）
- ピークメモリはタイルサイズに比例（ドキュメントサイズに非依存）
- フレームバジェット 32ms (~30fps)
- ターミナルリサイズ対応（ドキュメント再構築、スクロール位置保持）
- Effect パターン（モードハンドラが `Vec<Effect>` を返す。制御フローと副作用を分離）
  - viewport / session / effect の 3 モジュールに分離
- **フォーク分離**: Typst コンパイルは子プロセスで実行（メイン UI から隔離）

### セキュリティ

- **Landlock V4 サンドボックス**（Linux）: ファイルシステムアクセスを Git リポジトリルートまたは入力ファイルの親ディレクトリに制限
- フォント関連ディレクトリのホワイトリスト
- ネットワーク接続のデフォルト拒否（TCP bind/connect 不許可）
- V3 への graceful fallback、非対応環境ではスキップ（警告ログ）
- `--no-sandbox` で無効化可能
- リモート画像フェッチはデフォルト無効（`--allow-remote-images` で明示的にオプトイン）

### stdin 対応

```bash
cat README.md | mlux           # パイプ入力
curl https://example.com/doc | mlux  # curl パイプ
```

- バックグラウンドスレッドで stdin を読み取り、インクリメンタルに表示
- crossterm `use-dev-tty` により、stdin がパイプでもキーボード入力が使える

### CLI

```bash
# ビューアモード（デフォルト）
mlux <input.md> [--theme catppuccin]
mlux -                          # stdin から読む

# テーマ自動検出
mlux --theme auto <input.md>    # ターミナル背景色で dark/light 自動切替

# リモート画像を許可
mlux --allow-remote-images <input.md>

# PNG レンダリング
mlux render <input.md> -o <output.png> [--width 660pt] [--ppi 144] [--tile-height 500] [--theme catppuccin]

# デバッグ: フレームツリーダンプ
mlux render <input.md> --dump

# ファイル監視無効
mlux --no-watch <input.md>

# デバッグログ
mlux --log /tmp/mlux.log <input.md>         # ファイルにログ出力
mlux --debug <input.md>                     # debug レベルログ有効化
mlux --log /tmp/mlux.log --debug <input.md> # 両方
```

### 設定 (~/.config/mlux/config.toml)

```toml
theme = "auto"       # "auto" | "dark" | "light" | "catppuccin" | "catppuccin-latte"
width = 660.0        # ページ幅 (pt)
ppi = 144.0          # 解像度

[viewer]
scroll_step = 3      # j/k スクロール量
frame_budget = 32    # フレーム予算 (ms)
tile_height = 500.0  # タイル高 (pt)
sidebar_cols = 6     # サイドバー列数
evict_distance = 4   # LRU キャッシュ距離
watch_interval = 200 # ファイル監視間隔 (ms)
```

CLI 引数は設定ファイルを上書きし、ライブリロード時も保持される。

## mlux が明確に優位な領域

### 1. タイポグラフィ品質 — 圧倒的

Typst エンジンによる組版は ANSI ベースのツールでは原理的に不可能なレベル:
- カーニング、リガチャ、プロポーショナルフォント
- 両端揃え（ジャスティフィケーション）
- 日本語の禁則処理・行分割
- 見出し・本文・コードのフォントサイズの正確な制御
- テーブルのピクセル精度レイアウト

glow のテーブルは文字罫線 (`---||`) で描画されるため、セル内テキストの折り返しやアラインメントに限界がある。mlux の Typst テーブルは PDF 品質。

### 2. 日本語（CJK）サポート

IPAGothic + Noto Sans CJK JP のフォントチェーン、Typst の CJK 行分割アルゴリズムにより、日本語ドキュメントの表示品質は glow を大きく上回る。glow はワードラップが英語中心で、CJK の行分割はターミナルエミュレータ任せ。

### 3. 数式レンダリング — カテゴリ唯一

mitex 経由で LaTeX 数式を Typst ネイティブ数式に変換し、PDF 品質でレンダリングする。**ターミナル Markdown ビューアで数式をレンダリングできるのは mlux だけ**。glow, bat, mdcat, mcat いずれも非対応。インライン `$E = mc^2$` もディスプレイ `$$\int_0^\infty$$` も正しく組版される。

### 4. 画像・ダイアグラムの統合

ローカル画像をピクセル精度で Typst ドキュメントにインライン表示できる。さらに Mermaid ダイアグラムは SVG に自動レンダリングされ、コードブロックの代わりに図として表示される。glow は画像非対応、bat は画像非対応。mdcat は画像対応だがメンテナンス終了。

### 5. 検索ハイライト — ピクセル精度

ContentIndex による双方向マッピング（Markdown ソースバイト <-> Typst フレーム座標）を用いた**ワードレベルの検索ハイライト**。KGP オーバーレイ矩形で一致箇所をハイライトし、アクティブな一致をオレンジ色で区別する。less の全一致ハイライトに匹敵する視認性を、ピクセルレンダリング上で実現した。

### 6. シンタックスハイライト品質

tmTheme ベースのハイライトはピクセルレンダリングと組み合わさり、ANSI 16色/256色の制約がない。bat の syntect（Sublime 定義）に匹敵する言語対応を、PDF 品質の色忠実度で表示できる。

### 7. メモリ効率のアーキテクチャ

タイルベースの遅延レンダリング + LRU キャッシュは、巨大ドキュメントのスケーラビリティを保証する優れた設計。less の「ファイル全体を読まない」設計哲学と同じ精神を、ピクセルレンダリングという重い処理に対して実現している。コンテンツアドレスキャッシュにより、リビルド時に変更のないタイルはキャッシュヒットする。

### 8. Yank 機能の精密さ

ソースマッピング（Markdown バイト範囲 → Typst バイト範囲）による行単位・ブロック単位のヤンクはユニークな機能。コードブロックの特定行をピクセル座標からソースコード行に逆引きして OSC 52 でクリップボードにコピーする — これは他のどのツールにもない。

### 9. URL 操作とリンクナビゲーション

ソースマップを使った行 → URL の逆引きで、表示中のドキュメントからシームレスにリンクを開ける。`o` キー一発でカーソル行の URL を開き、`O` で全 URL を一覧してから選べる。さらに**ローカル `.md` ファイルへのタグジャンプ**が可能で、`Ctrl-O` で戻れるジャンプスタックにより、Markdown 文書間をブラウザライクに行き来できる。

### 10. テーマの拡張性

mlux のテーマは Typst ソースそのものなので、Typst の全機能（数式、図形、カスタムレイアウト）が使える潜在力がある。テーマ自動検出 (`--theme auto`) により、ターミナルの背景色に応じて dark/light テーマを自動切替する。

### 11. セキュリティアーキテクチャ

Landlock V4 サンドボックスとフォーク分離による防御的なアーキテクチャは、**ターミナル Markdown ビューアとしては異例のセキュリティレベル**。信頼できないドキュメントを開く際のリスクを大幅に軽減する。glow, bat, less にはこのようなサンドボックスは存在しない。

## mlux が劣る領域

### 1. ターミナル互換性

| ツール | 要求プロトコル | 対応ターミナル |
|--------|--------------|---------------|
| less | 基本 ANSI | ほぼ全て |
| glow | ANSI + TrueColor | ほとんど全て |
| **mlux** | **Kitty Graphics Protocol** | **Kitty, Ghostty, WezTerm のみ** |

設計上のトレードオフであり「弱点」とは言い切れないが、ユーザベースを大幅に制限する。Sixel フォールバックがあれば xterm/foot/mintty 等にも広がる。

### 2. パイプ入力の利便性

stdin 対応は実装されているが、less のように「どんなコマンドの出力でも受け取れる」状態とは異なる。ANSI 出力（例: `git log`）は Markdown として解釈されるため、一般 pager としての代替にはならない。これは設計上の必然であり欠点とは言い切れないが、less との差異ではある。

### 3. Markdown 対応要素の不足

タスクリスト、脚注、定義リストが未対応。glow は CommonMark + GFM の主要要素をほぼ全てカバーしている。ただし画像・数式・Mermaid は mlux が唯一の対応ツール。

### 4. 後方検索

less は `?` による後方検索、`&` によるフィルタリングが可能。mlux の検索は前方のみ（`/`）。ただしスマートケースや `Alt+1-9` ジャンプなど、インクリメンタル検索の操作性は高い。

## 結論

mlux は「ターミナルの文字セルの限界内で頑張る」という従来の Markdown ビューアの発想を超え、**ターミナル上の PDF ビューア**という新しいカテゴリを切り開いている。

**ページャとしての完成度** と **組版ビューアとしての完成度** は区別が必要:

- 組版ビューアとして: **97%** — 画像・数式・Mermaid・シンタックスハイライトすべて対応。ターミナル Markdown ビューアとして事実上の最高水準。残課題はタスクリスト・脚注程度
- ページャとして: **90%** — 検索ハイライト実装済み、TOC、タグジャンプ、ジャンプスタック完備。Sixel フォールバックが次の壁
- Markdown レンダラとして: **92%** — 画像・数式・Mermaid 対応で大幅改善。タスクリスト・脚注が残る
- セキュリティ: **85%** — Landlock V4 + フォーク分離。非 Linux 環境でのサンドボックス未対応

### ニッチとしての自己評価

mlux は「なんでもできる汎用 pager」ではない。しかし:

- **日本語技術ドキュメント** のコードブロック付きを読む
- **数式を含む論文やノート** をターミナルで閲覧する
- **Mermaid ダイアグラム** を含む設計ドキュメントを確認する
- **コードスニペットをヤンクして貼り付ける**
- **リンクを踏みながら .md ファイル間を行き来する**

という具体的なユースケースでは、less にも glow にも bat にも実現できない体験を提供できる。Kitty/Ghostty/WezTerm ユーザーにとっては、**既に「最良の Markdown ビューア」と評価できる水準**に達している。

### 次に最もインパクトのある改善候補

1. **Sixel フォールバック** — 対応ターミナルの大幅な拡大。現状の最大の普及障壁
2. **タスクリスト / チェックボックス** — GFM 互換性の向上。実装コストは低い
3. **脚注サポート** — 学術・技術文書での利便性向上
4. **後方検索 (`?`)** — less ユーザの期待に応える
5. **マルチファイルビューア** — 複数 .md ファイルのタブ切り替え
