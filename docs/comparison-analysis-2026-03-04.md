# mlux 完成度評価 — less / glow / 他ツールとの詳細比較

2026-03-04（前回: 2026-02-27 からの差分を反映）

## 根本的な設計思想の違い

まず認識すべきは、mlux が他のどのツールとも**根本的に異なるアプローチ**を取っていることである。

| ツール | レンダリング方式 | 出力 |
|--------|------------------|------|
| **less** | レンダリングなし（生テキスト） | 文字セル |
| **glow** | ANSI エスケープコード | 文字セル（装飾付き） |
| **bat** | シンタックスハイライト | 文字セル（装飾付き） |
| **mdcat** | ANSI + 画像プロトコル | 文字セル + インライン画像 |
| **mlux** | **Typst 組版エンジン → PNG** | **ピクセルレンダリング** |

glow や bat が「ターミナルの文字セルをどう装飾するか」という問題を解いているのに対し、mlux は「プロフェッショナルな組版結果をターミナルに表示する」という、一段上の問題を解いている。これは PDF ビューアに近い。

---

## 各ツールの概要

### less (Unix pager)

- レンダリングなし。Markdown をプレーンテキストとして表示
- 双方向スクロール、正規表現検索（`/`, `?`, `&`）、マーク、複数ファイル対応
- `F` キーで `tail -f` 相当のフォローモード
- パイプ入力 (`command | less`) に完全対応
- LESSOPEN/LESSCLOSE による入力プリプロセッサ拡張
- ほぼ全てのターミナルで動作。数十年の安定性

### glow (charmbracelet/glow)

- goldmark (CommonMark 準拠) でパース → Glamour で ANSI レンダリング
- **対応要素:** 見出し、段落、強調、取り消し線、インラインコード、フェンスコードブロック（chroma によるシンタックスハイライト）、引用、リスト（タスクリスト含む）、テーブル、リンク、水平線、定義リスト
- **非対応:** 画像（alt テキストのみ表示）、数式/LaTeX、Mermaid
- TUI モード（ファイルブラウザ）+ ページャモード
- `/` で前方検索、`n`/`N` で結果巡回
- テーマ: `ascii`, `auto`, `dark`, `dracula`, `tokyo-night`, `light`, `notty`, `pink` + カスタム JSON スタイルシート
- ANSI のみ（Kitty/Sixel 不要）、ほぼ全ターミナルで動作

### bat (sharkdp/bat)

- Markdown をレンダリングしない。ソースコードとしてシンタックスハイライト表示
- syntect (Sublime Text 定義) による 166+ 言語対応
- 行番号、Git diff インジケータ、自動ページング
- less をバックエンドページャとして使用可能

### mdcat (swsnr/mdcat) — 2025年1月アーカイブ済み

- ANSI レンダリング + iTerm2/Kitty/Sixel による画像インライン表示
- OSC 8 ハイパーリンク、シンタックスハイライト
- ページャ機能なし（`cat` 相当、パイプで less に渡す）
- **メンテナンス終了**

### mcat (Skardyy/mcat)

- 画像・動画・PDF・DOCX・Markdown のマルチフォーマットビューア
- Sixel / Kitty Graphics Protocol による画像表示
- 比較的新しいプロジェクト

## mlux の現在の機能セット（2026-03-04 時点）

### Markdown 変換 (convert.rs)

**対応ブロック要素:**
- 見出し (h1-h6、h4-h6 も正式に styled)
- 段落（ソフト改行・ハード改行）
- フェンスコードブロック（言語指定付き、シンタックスハイライト対応）
- 順序なしリスト / 順序付きリスト
- 引用（ネスト対応、深さ上限 10）
- テーブル（列数・セル内容）
- 水平線

**対応インライン要素:**
- 太字、イタリック、取り消し線
- インラインコード（バッククォート含むケースは `#raw()` 関数使用）
- リンク（空 URL は平文に降格）
- Typst 特殊文字エスケープ (`#*_<>@$\\/~()[]`)

**非対応:**
- 画像
- タスクリスト / チェックボックス
- 脚注
- 定義リスト
- 上付き / 下付き文字
- インライン HTML
- YAML フロントマター

### レンダリング (render.rs, world.rs)

- Typst 0.14 組版エンジンによる PDF 品質のレンダリング
- 設定可能パラメータ: width (660pt), PPI (144), tile-height (500pt), theme
- フォント: IPAGothic → Noto Sans CJK JP → Noto Sans
- テーマ: Typst ソースファイル（Catppuccin Mocha 同梱）

### ★ シンタックスハイライト（2026-03-04 新規追加）

- **tmTheme ベースのシンタックスハイライト** が有効化
- Catppuccin Mocha 用 tmTheme ファイル（2100行超）をバーチャルファイルとして埋め込み
- Typst の `#set raw(theme: ...)` 機能を活用
- 言語指定付きコードブロック（` ```rust ` 等）で意味ベースのカラーリングが実現
- glow（chroma）・bat（syntect）と並ぶ品質。さらにピクセル精度の色表現が可能

### ターミナルビューア (viewer.rs, tile.rs)

**表示:**
- Kitty Graphics Protocol によるピクセルレンダリング
- 6 カラムサイドバーに行番号表示
- ステータスバー（ファイル名、スクロール位置、パーセンテージ、フラッシュメッセージ）
- 非同期ビルド + ローディング画面（100ms 以上かかる場合）、ビルド中断対応（`q`/`Esc`/`Ctrl-C`）

**モード構成（4モード）:**

```
Normal ─(/)→ Search ─(Enter)→ Normal
       ─(:)→ Command ─(Enter/Esc)→ Normal
       ─(O)→ URL Picker ─(Enter/Esc)→ Normal
```

**ナビゲーション (vim/less ライク — Normal モード):**

| キー | 動作 |
|------|------|
| `j` / `↓` | 下スクロール（`Nj` で N ステップ） |
| `k` / `↑` | 上スクロール（`Nk` で N ステップ） |
| `d` | 半ページ下（`Nd` 対応） |
| `u` | 半ページ上（`Nu` 対応） |
| `g` | 先頭へ / `Ng` で N 行目へジャンプ |
| `G` | 末尾へ / `NG` で N 行目へジャンプ |
| `y` | 行ヤンク（`Ny` で N 行目のコードをコピー） |
| `Y` | ブロックヤンク（`NY` で N 番目のブロック全体をコピー） |
| `/` | 検索モードへ |
| `o` | カーソル行の最初の URL をブラウザで開く（`No` で N 行目） |
| `O` | URL ピッカーモードへ（全 URL 一覧） |
| `:` | コマンドモードへ |
| `Esc` | 数値入力キャンセル |
| `q` / `Ctrl+C` | 終了 |

**★ 検索モード（2026-02-27 以降に実装）:**
- `/` で起動
- Markdown ソースに対する正規表現検索
- `j`/`k` で結果巡回
- `Enter` で該当行へジャンプし Normal モードに戻る
- Normal モードから `n`/`N` で次/前の一致へジャンプ

**★ URL ピッカーモード（2026-02-28 新規追加）:**
- `O` または `:open` で起動
- 文書内の全 URL を一覧表示（Markdown リンクテキスト付き）
- 裸の `https://` URL も自動抽出
- `j`/`k` で選択、`Enter` でブラウザ起動

**コマンドモード:**
- `:reload` / `:rel` — 設定とドキュメントを再読み込み
- `:quit` / `:q` — 終了
- `:open` — URL ピッカーモードへ

**アーキテクチャ:**
- タイルベース遅延レンダリング（ドキュメント全体を 1 回コンパイル → フレームツリーを垂直タイルに分割）
- 可視タイルのみ PNG 化、LRU キャッシュ（前後 ±4 タイル）
- プリフェッチワーカー（前方 2 + 後方 1 タイル先行レンダリング）
- ピークメモリはタイルサイズに比例（ドキュメントサイズに非依存）
- フレームバジェット 32ms (~30fps)
- ターミナルリサイズ対応（ドキュメント再構築、スクロール位置保持）
- Effect パターン（モードハンドラが `Vec<Effect>` を返す。制御フローと副作用を分離）

### ★ stdin 対応（2026-03-01 新規追加）

```bash
cat README.md | mlux           # パイプ入力
curl https://example.com/doc | mlux  # curl パイプ
```

- バックグラウンドスレッドで stdin を読み取り、インクリメンタルに表示
- crossterm `use-dev-tty` により、stdin がパイプでもキーボード入力が使える

### CLI

```bash
# ビューアモード（デフォルト）
mlux <input.md> [--theme catppuccin]
mlux -                          # stdin から読む

# PNG レンダリング
mlux render <input.md> -o <output.png> [--width 660pt] [--ppi 144] [--tile-height 500] [--theme catppuccin]

# デバッグ: フレームツリーダンプ
mlux render <input.md> --dump

# ファイル監視無効
mlux --no-watch <input.md>

# デバッグログ（ターミナル UI を汚さないようファイルに書く）
mlux --log /tmp/mlux.log <input.md>
```

## mlux が明確に優位な領域

### 1. タイポグラフィ品質 — 圧倒的

Typst エンジンによる組版は ANSI ベースのツールでは原理的に不可能なレベル:
- カーニング、リガチャ、プロポーショナルフォント
- 両端揃え（ジャスティフィケーション）
- 日本語の禁則処理・行分割
- 見出し・本文・コードのフォントサイズの正確な制御
- テーブルのピクセル精度レイアウト

glow のテーブルは文字罫線 (`─│┌┐`) で描画されるため、セル内テキストの折り返しやアラインメントに限界がある。mlux の Typst テーブルは PDF 品質。

### 2. 日本語（CJK）サポート

IPAGothic + Noto Sans CJK JP のフォントチェーン、Typst の CJK 行分割アルゴリズムにより、日本語ドキュメントの表示品質は glow を大きく上回る。glow はワードラップが英語中心で、CJK の行分割はターミナルエミュレータ任せ。

### 3. シンタックスハイライト品質

tmTheme ベースのハイライトはピクセルレンダリングと組み合わさり、ANSI 16色/256色の制約がない。bat の syntect（Sublime 定義）に匹敵する言語対応を、PDF 品質の色忠実度で表示できる。

### 4. メモリ効率のアーキテクチャ

タイルベースの遅延レンダリング + LRU キャッシュは、巨大ドキュメントのスケーラビリティを保証する優れた設計。less の「ファイル全体を読まない」設計哲学と同じ精神を、ピクセルレンダリングという重い処理に対して実現している。

### 5. Yank 機能の精密さ

ソースマッピング（Markdown バイト範囲 → Typst バイト範囲）による行単位・ブロック単位のヤンクはユニークな機能。コードブロックの特定行をピクセル座標からソースコード行に逆引きして OSC 52 でクリップボードにコピーする — これは他のどのツールにもない。

### 6. URL 操作の統合

ソースマップを使った行 → URL の逆引きで、表示中のドキュメントからシームレスにリンクを開ける。`o` キー一発でカーソル行の URL を開き、`O` で全 URL を一覧してから選べる。ページャとしての実用性が高い。

### 7. テーマの拡張性

mlux のテーマは Typst ソースそのものなので、Typst の全機能（数式、図形、カスタムレイアウト）が使える潜在力がある。

## mlux が劣る領域

### 1. 検索の実用性

**[改善済]** 前回報告時は「検索なし」だったが、現在は正規表現検索モードが実装されている。ただし ANSI ベースツールとの比較ではまだ差がある:

| ツール | 検索方式 | ハイライト |
|--------|----------|-----------|
| less | 正規表現（前方/後方/フィルタ） | 全一致をハイライト表示 |
| glow | テキスト前方検索 | 一致箇所を強調 |
| **mlux** | 正規表現（Markdown ソース）| 一致行へジャンプのみ（一致箇所の視覚ハイライトなし） |

mlux の検索はソースに対するテキスト検索であり、レンダリング結果のピクセル上への検索マーカー描画が未実装。「どこがヒットしたか」が視覚的にわかりにくい点は残課題。

### 2. ターミナル互換性

| ツール | 要求プロトコル | 対応ターミナル |
|--------|--------------|---------------|
| less | 基本 ANSI | ほぼ全て |
| glow | ANSI + TrueColor | ほとんど全て |
| **mlux** | **Kitty Graphics Protocol** | **Kitty, Ghostty, WezTerm のみ** |

設計上のトレードオフであり「弱点」とは言い切れないが、ユーザベースを大幅に制限する。Sixel フォールバックがあれば xterm/foot/mintty 等にも広がる。

### 3. 画像サポートなし

Markdown の `![alt](url)` を処理していない。ただし glow も画像は表示できず、画像を表示できるのは mdcat（アーカイブ済み）と mcat 程度。業界全体の課題であり mlux 固有の弱点ではない。

### 4. パイプ入力の利便性

**[部分的に改善済]** stdin 対応は実装されたが、less のように「どんなコマンドの出力でも受け取れる」状態とは微妙に異なる。ANSI 出力（例: `git log`）は Markdown として解釈されるため、一般 pager としての代替にはならない。これは設計上の必然であり欠点とは言い切れないが、less との差異ではある。

### 5. Markdown 対応要素の不足

タスクリスト、脚注、定義リスト、画像が未対応。glow は CommonMark + GFM の主要要素をほぼ全てカバーしている。

## 結論

mlux は「ターミナルの文字セルの限界内で頑張る」という従来の Markdown ビューアの発想を完全に超え、**ターミナル上の PDF ビューア**という新しいカテゴリを切り開いている。

**2026-03-04 時点の完成度は、2 月末時点からさらに大きく向上している。** 前回評価で「最大の弱点」とされた検索がついに実装され、コードハイライトも glow と同等水準に達した。stdin 対応でページャとしての汎用性も高まった。

ただし **ページャとしての完成度** と **組版ビューアとしての完成度** は区別が必要:

- 組版ビューアとして: **93%** — シンタックスハイライトが加わり、コードドキュメントの表示品質が大幅向上。残課題は画像・数式
- ページャとして: **82%** — 検索・stdin が加わり実用性が大幅向上。検索結果の視覚ハイライト、Sixel フォールバックが次の壁
- Markdown レンダラとして: **80%** — 前回から変化なし。画像・タスクリスト・脚注が未対応

### ニッチとしての自己評価

mlux は「なんでもできる汎用 pager」ではない。しかし:

- **日本語技術ドキュメント** のコードブロック付きを読む
- **コードスニペットをヤンクして貼り付ける**
- **リンクを踏みながら調査する**

という具体的なニッチでは、less にも glow にも bat にも実現できない体験を提供できる。Kitty/Ghostty/WezTerm ユーザーにとっては、既に「常用に値する」水準に達していると評価できる。

### 次に最もインパクトのある改善候補

1. **検索結果のピクセルハイライト** — 検索モードは機能するが、一致箇所のビジュアルフィードバックが弱い。ソースマップから一致行の座標を計算し、オーバーレイ矩形を描画する
2. **Sixel フォールバック** — 対応ターミナルの大幅な拡大。現状の最大の普及障壁
3. **画像レンダリング** — `![alt](url)` のフェッチ + Kitty Protocol 表示。実現するとカテゴリ随一の機能になる
4. **数式レンダリング** — Typst が得意とする LaTeX 数式。他のどのターミナルビューアも実現していない
